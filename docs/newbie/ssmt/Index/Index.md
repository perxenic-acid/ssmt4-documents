# SSMT

**SSMT4** 是一款基于 **3Dmigoto** 的综合 Mod 工具箱，整体定位是 **简单易用**。

目前项目基于 **Rust + Tauri** 架构开发。

![alt text](image.png)

## 发展历史

SSMT 系列的演进过程如下：

- **SSMT4** 前身是 **SSMT3**，使用 **WinUI3** 开发
- **SSMT3** 前身是 **DBMT**，前期使用 **WinForm** 架构，后期使用 **WinUI3** 开发
- **DBMT** 前身是 **MMT**，使用 **C++ 内核 + WinForm** 开发
- **MMT** 前身是 **NMBT**，使用 **C++ 内核 + Python 界面** 开发
- **NMBT** 前身是 **3Dmigoto-Wheel**，它是一个使用 `Json` 文件配置后即可直接运行的 **C++ 内核 exe 控制台程序**
- **3Dmigoto-Wheel** 的前身是 **3Dmigoto-Tailor**，一个综合性的 **Python 脚本工具集**

> SSMT 的第一个版本诞生于 2022 年，至今已持续更新和开发运维 4 年时间。

## 设计哲学

### 简单

在支持复杂使用场景的前提下，尽可能把设计做到极致简单。

### 优雅

颜值必须高，用着必须爽。

### 统一

多个游戏复用一套流程，免去学习大量不同工具的成本。

## 天生支持将 Mod 导入 Blender 二次编辑，用于修复旧 Mod

> 我一般称之为 **Mod 逆向**，但其本质并非“逆向”，而是对 Mod 文件做一次 **数据格式转换**，使其能够被导入到 **Blender** 中。

SSMT 系列工具从第一个版本诞生至今，每个版本都天生支持 **Mod 逆向**。

SSMT 基于数据类型的架构，可以同时支持 **正向提取** 和 **逆向提取**，并且配置可以自由增删。

在其它游戏中，Mod 模型通常都可以较轻松地提取出来进行修复，从而处理那些在版本更新后损坏的旧 Mod；而 **3Dmigoto** 类 Mod 的转换格式相对更复杂，SSMT 系列工具正是为了解决这个问题。

**Mod 逆向** 功能也正是为了修复旧 Mod 而诞生，并在后续过程中逐渐完善。
