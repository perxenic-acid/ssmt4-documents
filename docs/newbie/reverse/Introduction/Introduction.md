# 自动逆向功能

如果你手上的 **3Dmigoto 格式 Mod** 需要还原模型，而你又不想手动硬啃复杂的 `ini`，那么 **自动逆向功能** 的价值就在于：**拿到 Mod 文件后，无需开启游戏，也能直接分析并还原模型数据**。

它的工作原理，是对 `ini` 文件中的语法进行全自动分析，并构建抽象的逻辑分析链完成语法解析，从 `ini` 内容中识别出 **Buffer 文件** 可能组合成的模型文件。

## 你能得到什么

- **无需开启游戏**：只要获取到 Mod 文件，就能通过分析 `ini` 的方式，一键将 Mod 中的模型逆向还原为 Mod 文件。
- **复杂 `ini` 自动解析**：支持复杂 `ini` 解析，并模拟 **3Dmigoto** 的解析逻辑。
- **还原速度快**：核心架构使用 **Rust** 编写，响应速度快，通常可在 **3 秒内**还原模型。
- **尽量完整的数据还原**：可以将所有可能的数据类型尽量完整地逆向出来，以确保逆向结果的正确性。
- **兼容性高**：基本上 **99%** 的 Mod 都能逆向；如果配合手动修改 `ini` 以及手动逆向，则所有支持类型的开源格式 Mod 都能逆向。

## 这类功能为什么有价值

对于结构简单的 Mod，手动处理还不算太难；但一旦遇到复杂 `ini`、多分支切换，甚至几千行逻辑的 Mod，手动逆向的时间成本和出错成本都会明显上升。

自动逆向功能的意义，就是把这部分原本繁琐、重复、容易出错的工作尽可能自动化完成。
